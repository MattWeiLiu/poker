# High-Performance Poker Equity Evaluation Engine

A ultra-optimized Texas Hold'em equity calculation tool that combines **C++ Lookup Table (LUT)** technology with **Python multiprocessing** for millisecond-level full-spectrum hand analysis.

---

## 🎲 Mathematical Foundation

### Variable Definitions
| Description | Symbol |
| :--- | :---: |
| Total Participants (including yourself) | $$n$$ |
| Dead Money (Current Pot) | $$P$$ |
| Call Amount (To Call) | $$B$$ |
| Win Rate | $$p$$ |
| Loss Rate | $$q$$ |
| Draw Rate | $$d$$ |

> [!NOTE]
> $d = 1 - (p + q)$

### Profit Matrix (Win Condition)
This matrix describes the net profit generated by different action paths after the decision point. Excluding sunk costs, the **Fold** action is set as the baseline ($0$).

| MINE \ Oppo | Calls | Folds |
| :---: | :---: | :---: |
| **Calls** | $$(n-1)B + P$$ | $$P$$ |
| **Folds** | $$0$$ | $$0$$ |

### Expected Value (EV) Matrix
| MINE \ Oppo | Calls | Folds |
| :---: | :---: | :---: |
| **Calls** | $$p((n - 1)B + P) - qB + d\frac{P}{n}$$ | $$P$$ |
| **Folds** | $$0$$ | $$0$$ |

## 🚀 Core Optimization Technologies

- **phevaluator Integration**: Utilizes a Perfect Hash algorithm, reducing a 7-card evaluation to merely **~1.8μs**.
- **ctypes Direct Memory Access**: Leverages C APIs to bypass Python’s performance bottlenecks, achieving microsecond-level function calls.
- **Integer-based API**: Eliminates string concatenation and parsing; data is passed between Python and C++ as primitive integer IDs.
- **Isomorphism Compression**: Simplifies 1,326 starting hand combinations into 169 representative groups, drastically reducing the computational workload.
- **Parallel Processing**: Automatically detects CPU core count and executes concurrent calculations, scaling linearly with hardware capacity.

## 📂 Project Structure

- `PokerFullSpectrum.py`: Main engine responsible for full-spectrum recursive calculations and aggregation for 169 hand groups.
- `PokerQuery.py`: Query tool supporting hand normalization and equity retrieval from pre-calculated results.
- `poker_eval_ctypes.py`: Python ctypes wrapper supporting macOS (`.dylib`) and Linux (`.so`).
- `evaluator/`: C++ evaluator source tree.
  - `poker_eval_api.cpp`: C-interface bridge for Python calls.
  - `phevaluator/`: Core Lookup Table (LUT) library.
- `cards.yaml`: Global configuration for cards and ranks.
- `results/`: Output directory for calculation results (CSV format).

## 🛠️ Setup & Compilation

High-performance binary compilation is required for optimal results.

### 1. Prerequisite Installation (Linux Example)
```bash
sudo apt update && sudo apt install -y build-essential python3-pip
pip3 install pyyaml
```

### 2. Compiling the Core Library
```bash
# Compile phevaluator static library
cd evaluator/phevaluator/cpp
make libpheval.a

# Compile bridge library
cd ../..
g++ -std=c++17 -O2 -shared -fPIC -I./phevaluator/cpp/include -L./phevaluator/cpp -o libpoker_eval.so poker_eval_api.cpp ./phevaluator/cpp/libpheval.a
```
> [!TIP]
> On **macOS**, ensure the output dynamic library is named `libpoker_eval.dylib`.

## 📈 Usage Guide

### Full Spectrum Calculation
```bash
python3 PokerFullSpectrum.py
```
This iterates through 169 starting hand combinations. Each hand takes approximately 1-2 minutes depending on CPU performance. Results are stored in the `results/` folder.

### Query Hand Equity
Use `PokerQuery.py` for automatic normalization and lookup:
```bash
# Query Pre-flop Equity
python3 PokerQuery.py AH KD

# Query Post-flop (Flop stage)
python3 PokerQuery.py AH KD 2S 3H 4D
```
